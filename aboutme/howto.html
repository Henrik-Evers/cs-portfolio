YEEETI created an HTML webpage that tells the user some facts about me, in addition to a summary of what I have learned in the Introduction to Programming class. The project uses coordinated HTML (Hypertext Markup Language), CSS (Cascading Style Sheets), and JS (JavaScript) files to work.

How I Made It

YEEETTo connect the CSS file to HTML, I used the following code: <link rel="stylesheet" type="text/css" href="aboutme.css"/> The `type` and `rel` attributes of the <link> tag tells HTML that the linked file is a CSS stylesheet. The `href` attribute points to the location of the file.
YEEETConecting a JS file is a little different. for this, I used the following code inside of the <head> tag:
1: <script src="aboutme.js">
2: /* The code */
3: </script>
In Line 1, the `src` attribute points to the location of the JavaScript file. Line 2 represents where the code would go (the /* and */ are used to open and close comments in JS). From there, I could call a function present in `aboutme.js`. For example, the following picture shows the function navButtonClicked from aboutme.js:

Now, if I write navButtonClicked(); in the <script> tag mentioned earlier, it would act as if the user clicked the `Pages` button when they load the page. You can see the function in action by pressing the button `Pages` button in the top left.

Navigation Buttons

However, simply adding some JavaScript will not make our buttons look how we want them to (actually, it can using the JSDOM, but that isn't needed in these circumstances). To do that, the CSS on the right was added to `aboutme.css`. Some of the things, like height, width, background-color, and border-color attributes are pretty self explanatory. However, some of them are not that way.
  • position:fixed; makes the element go wherever I tell it to, regardless of where it would normally go. The position never changes, even if you scroll. The two attributes below it, top:0px; and left:0px; tell the element to be zero pixels from the top and zero pixels from the left, placing it in the top left corner (where the navbuttons are).
  • border-radius:5px; sets the radius of the curve in the corners to be 5 pixels. If you open the navbutton (navigation button) dropdown, you can see the curve at the bottom corners. This makes them softer and easier on the eyes than a sharp 90 degree angle in the corners.
  • transition-duration:0.4s; makes the element take four tenths of a second to perform a transition. In the navButtonClicked function that was discussed above, this would mean it takes 0.4 seconds for the button to move to its new position.
  • z-index:10; sets the z-index to 10. All elements have a default z-index of 0. When two elements overlap, the element with a higher z-index is shown. If you think of z-index as distance from sea level, and you are looking down from space, the higher object will be on top of the lower object. Therefore, an element with a z-index of 10 will shown over an element with a z-index of 0. I set the z-index to 10 to make sure it was above the other elements. Although I could have used a lower z-index, 10 makes it easy to expand the website in the future.
